This folder contains the "thesis-related" versions of scripts and functions for the neuronal class-based evolutionary pipeline. Functions are required for circuits' activity simulation and evolution simulation. Scripts are typically more ad hoc and situative, for analysis, visualization and so on. 
Here I will explain the most important pieces of code.

full_evo_sim.m 
The main simulation function that runs one "regular" (with no environmental feedback) evolution simulation. Takes very few input arguments, a lot of parameters are hardcoded and have to be changed within the function itself (probably later will be replaced by a config files).
Important settings within the function:
- Initial circuit of Net2 format. Can be either uploaded as a separate .mat file with a line like "load('MyNet2_I_111_O_1_NotEvolved.mat');", or generated by a special function like "Net2=sleep_circuit_build;". The name of the loaded file encodes the input-output structure of the circuit, it must match the protocol maker function.
- Zoo.Settings.PreMutationsN=50; This variable shows how many times the mutation routine is applied to the initial circuit before the beginning of selection process. Set to 0 if you work with exported or designed circuit that you want unchanged. Set to 50 or more if you want a "random", "non-functional" circuit.
- Gen_N=300; Inst_N=50; How many generations will be simulated, and how many circuits within each generation.
- Protocol{C}=Protocol_maker_...  A family of "Protocol_maker_..." functions creates I-O sequences. The function and the I-O structure of the initial circut mast match, otherwise it is not clear how to apply the I-O sequence generated. Check the I-O structure required by "Protocol_maker_..." inside of its code
- offsprings=offspring_allocation(winner_order,"proportional");  The offspring allocation function. Decides who has how many offspring based on the fitness values. Has two modes now: "proportional" for the proportional allocation, and "winner,x" for "winner-takes-all" allocation with x best circuits having equal number of offspring.

full_evo_sim_envelope.m 
This is a script that allows iterative runs of full_evo_sim.m with the same parameters. If multiple independent evolution simulations are required (like in the examples 1, 3, 4, 5 of the thesis)

legality.m
This function checks the structure of the Net2 variable. Either returns a report that the "genotype" is legal (i.e. all the traits are defined, there are no unattributed traits, and so on), or a report about mismatches.
Mismatches may be of sorts like "there are six subclasses, but the refractory period is defined only for five", or "there is a plasticity within synapses of non-existant pair of subclasses". 
The mutation routine is streamlined so that the mutation of a legal "genotype" is always legal, so there is no point to check legality during the evolution loop. It is, however, checked at the beginning (although all the circuits from the thesis are legal and can be used with no hesitation).
However, if one manually designes or alters a Net2 variable, the legality check is highly advised: I myself usually fail to design a legal Net2 from the first attempt.
